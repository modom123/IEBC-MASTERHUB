③ 🚀 Deploy for 20 Consultants — GitHub Pages

Host this hub on GitHub Pages — version-controlled, always at the same permanent URL, free forever. Every consultant bookmarks one link and it never changes.

⑴ First Time Setup
1. Go to github.com → Sign in (or create account)
2. Click + New repository
3. Name it iebc-hub → set to Public → Create
4. Click uploading an existing file
5. Drag in this HTML file → rename it index.html
6. Click Commit changes
7. Go to Settings → Pages → Source: Deploy from branch → Branch: main → folder: / (root) → Save
8. GitHub gives you: https://yourusername.github.io/iebc-hub
9. Share that URL with all 20 consultants — done.
⑵ Updating the Hub (Any Future Change)
1. Go to github.com/yourusername/iebc-hub
2. Click index.html → click the pencil ✏️ icon (Edit)
3. Delete all content → paste in the new hub file contents
4. Click Commit changes
5. GitHub redeploys in ~60 seconds — same URL, no disruption

Or via file upload:
Click Add file → Upload files → drop the new index.html → Commit → live in 60s
⚠️ The file MUST be named index.html — not the original filename
⑶ Custom Domain (Optional)
1. Buy iebc.io on Namecheap (~$12/yr)
2. GitHub repo → Settings → Pages → Custom domain
3. Enter hub.iebc.io → Save
4. In Namecheap DNS: add a CNAME record pointing hub → yourusername.github.io
5. Check Enforce HTTPS — free SSL auto-enabled
6. Consultants now go to hub.iebc.io — professional and permanent
⑷ Why GitHub vs Netlify
✅ URL never changes — updates replace the file, not the site
✅ Full version history — roll back any bad update in 1 click
✅ Free forever — no plan tiers, no bandwidth limits
✅ No drag-and-drop required — edit directly in browser
✅ GitHub Actions optional — auto-deploy on every save if you want CI/CD later
💡 Consultant onboarding: Send each consultant the GitHub Pages URL. They open it → enter Supabase credentials once → bookmark it. All 20 consultants sync to the same Supabase database in real time. When you update the hub, they refresh and get the new version instantly — no re-setup needed.
